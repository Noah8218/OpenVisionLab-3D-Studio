# Assimp zlib Core Provenance

Date: 2026-07-16

## Purpose

This document records a bounded compiler-input provenance check for the zlib
core that the fixed Open3D `0.19.0` clean Release build compiles through Assimp
`v5.4.2`. It is evidence for the fixed source/delta/current-build slice only.
It is not a binary-reproducibility, notice, redistribution, product-integration,
or Viewer/Runner-parity claim.

## Fixed Inputs

The Assimp update commit is the checked import boundary. It is an ancestor of
the fixed Assimp tag. The source path history after that commit is empty for
each of the 25 compiler-read source files even though later commits touch other
files under `contrib/zlib`.

## Checked Source Set

The fixed closure has 15 C compilation units, 9 private headers, and `zlib.h`:

`adler32.c`, `compress.c`, `crc32.c`, `crc32.h`, `deflate.c`, `deflate.h`,
`gzclose.c`, `gzguts.h`, `gzlib.c`, `gzread.c`, `gzwrite.c`, `infback.c`,
`inffast.c`, `inffast.h`, `inffixed.h`, `inflate.c`, `inflate.h`,
`inftrees.c`, `inftrees.h`, `trees.c`, `trees.h`, `uncompr.c`, `zlib.h`,
`zutil.c`, and `zutil.h`.

The verifier checks every file at four points: official zlib `v1.2.13`, the
Assimp import, Assimp `v5.4.2`, and the clean-build source directory. All 25
Git blobs and exported/build-file hashes agree.

Assimp CMake builds `zlibstatic` from the 15 listed sources, 9 listed private
headers, and `zlib.h`. `zlib.h` includes `zconf.h`; the latter is generated by
Assimp's `configure_file(zconf.h.cmakein ... @ONLY)` step. The generated header
is intentionally outside this upstream source-identity subset. Its generation
is recorded so this document does not imply that all effective build inputs are
unmodified upstream zlib files.

## Reproduction

Run from the repository root with a previously unused working directory:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\verify-assimp-zlib-provenance.ps1 `
  -WorkingDirectory artifacts\dependency-candidates\assimp-zlib-provenance-20260716\manual-work `
  -ReportPath artifacts\dependency-candidates\assimp-zlib-provenance-20260716\manual-report.json
```

Expected positive output:

```text
AssimpZlibProvenance|Pass|zlib=04f42ceca40f73e2978b50e93806c2a18c1281fc|assimp=ddb74c2bbdee1565dda667e85f0c82a0588c8053|files=25|history=25|issues=0
```

The post-fix local report is
`artifacts/dependency-candidates/assimp-zlib-provenance-20260716/zlib-provenance-rerun.json`.

## Fail-Closed Check

Passing an incorrect expected upstream revision records a failure and returns
exit code `1`:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\verify-assimp-zlib-provenance.ps1 `
  -ExpectedZlibRevision 0000000000000000000000000000000000000000 `
  -WorkingDirectory artifacts\dependency-candidates\assimp-zlib-provenance-20260716\negative-work `
  -ReportPath artifacts\dependency-candidates\assimp-zlib-provenance-20260716\negative-report.json
```

The controlled local negative run returned `1` with one recorded issue.

## Claim Boundary

This evidence establishes all of the following for this fixed build:

- The official zlib `v1.2.13` tag resolves to the recorded commit.
- All 25 fixed source inputs are byte-identical at upstream, Assimp import,
  Assimp `v5.4.2`, and clean-build source locations.
- Each fixed source path has no post-import Assimp history through `v5.4.2`.
- The actual CMake source/header lists and the existing compiler-read closure
  agree on the 25-file subset.
- The generated `zconf.h` boundary is explicit rather than silently treated as
  an upstream source file.

It does not establish upstream release-signature or ownership identity,
complete zlib-history provenance, third-party notice disposition, byte-identical
binary reproduction, redistribution approval, product adoption, inspection
metric correctness, or physical metrology accuracy.
